You have an Azure subscription that contains a virtual network named VNet1. VNet1 uses an IP address space of 10.0.0.0/16 and contains the VPN Gateway and subnets in the following table:

Subnet1 contains a virtual appliance named VM1 that operates as a router.
You create a routing table named RT1.
You need to route all inbound traffic from the VPN gateway to VNet1 through VM1.
How should you configure RT1? To answer, select the appropriate options in the answer area.

Correct Answer: Box 1: 10.0.0.0/16 Address prefix destination-> Vnet 1 (Address space of Vnet1) Box 2: Virtual appliance Next hop type VM1 ->Virtual Appliance. You can specify IP address of VM 1 when configuring next hop as Virtual appliance. Box 3: Gateway Subnet Assigned to This route is to be followed by Gateway Subnet for the incoming traffic. You can associate routing table to the Subnet from Rout Table -> subnet ->Associate.
