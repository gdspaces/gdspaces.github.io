Your company's Azure solution makes use of Multi-Factor Authentication for when users are not in the office. The Per Authentication option has been configured as the usage model.
After the acquisition of a smaller business and the addition of the new staff to Azure Active Directory (Azure AD) obtains a different company and adding the new employees to Azure Active Directory (Azure AD), you are informed that these employees should also make use of Multi-Factor Authentication.
To achieve this, the Per Enabled User setting must be set for the usage model.
Solution: You create a new Multi-Factor Authentication provider with a backup from the existing Multi-Factor Authentication provider data.
Does the solution meet the goal?

  • A. Yes
  • B. No

It is not necessary to create a new Multi-Factor Authentication provider in this scenario. Instead, you can simply change the usage model from Per Authentication to Per Enabled User. This will enable Multi-Factor Authentication for all users in Azure AD, including the newly added employees. To change the usage model, follow these steps: Sign in to the Azure portal as a global administrator. Navigate to Azure Active Directory > Security > MFA. Click on the settings link at the top of the page. Under the usage model section, select the Per Enabled User option. Click on the Save button to apply the changes. Once the usage model has been updated, all users in Azure AD, including the new employees, will be required to use Multi-Factor Authentication. It is not necessary to create a new Multi-Factor Authentication provider, as the existing provider will continue to work with the updated usage model.
