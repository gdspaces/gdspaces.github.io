Your company's Azure subscription includes Azure virtual machines (VMs) that run Windows Server 2016.
One of the VMs is backed up every day using Azure Backup Instant Restore.
When the VM becomes infected with data encrypting ransomware, you are required to restore the VM.
Which of the following actions should you take?

  • A. You should restore the VM after deleting the infected VM.
  • B. You should restore the VM to any VM within the company's subscription.
  • C. You should restore the VM to a new Azure VM.
  • D. You should restore the VM to an on-premise Windows device.

The correct answer is C. You should restore the VM to a new Azure VM. In the scenario where a VM is infected with ransomware, it's best practice to isolate and remove the infected VM from the network as soon as possible. Once the infected VM has been isolated and deleted, you can initiate a restore process using Azure Backup Instant Restore. The restore process should create a new VM with the restored data. Option A is incorrect because restoring data to the same VM is not recommended if it was infected with ransomware. The restore process should create a new VM with the restored data. Option B is incorrect because you can't restore a VM to any VM within the company's subscription. You can only restore a VM to the original VM or a new Azure VM. Option D is also incorrect because Azure Backup is a cloud-based backup solution, and it doesn't support restoring VMs to on-premise Windows devices.

C. You should restore the VM to a new Azure VM. In the scenario where a virtual machine is infected with ransomware, it is important to isolate the compromised virtual machine to prevent the spread of the ransomware. Restoring the virtual machine after deleting the infected VM is not recommended, as it is possible that the backup of the infected VM may also be compromised. The recommended approach is to restore the VM to a new VM, as this provides a clean environment that is not compromised by the ransomware. Additionally, restoring the VM to an on-premises Windows device is not recommended since this may compromise the device as well.
